## ExploreWithMe

Эта часть проекта представляет собой микросервисную, она предназначена для управления событиями, пользователями, запросами на участие, комментариями и статистикой

## Архитектура

### Выделенные сервисы

Проект разделен на несколько микросервисов, вот их список:

1. **stats-server**:
    - хранит и предоставляет статистику по просмотрам событий

2. **comment-service**:
    - управляет комментариями к событиям: создание, обновление, удаление, получение списков

3. **event-service**:
    - управляет событиями: создание, обновление, публикация, получение списков (админ, приватный, публичный апи)

4. **request-service**:
    - обрабатывает запросы на участие в событиях: создание, отмена, подтверждение/отклонение

5. **user-service**:
    - управляет пользователями: создание, удаление, получение списков

### Инфраструктурные компоненты

- **discovery-server** (Eureka Server): обнаружение сервисов, все микросервисы регистрируются здесь для динамического обнаружения
    - Порт: 8761.
- **config-server**: централизованное управление конфигурациями, загружает ямл-конфиги для каждого сервиса из Git-репозитория или локальных файлов
    - Порт: 8888.
- **gateway-server** (Spring Cloud Gateway): шлюз для маршрутизации внешних запросов к сервисам, использует Eureka для маршрутизации по именам сервисов
    - Порт: 8080.
    - Маршруты: определяют пути для каждого сервиса (например, `/admin/events/**` → event-service)

### Взаимодействие между сервисами

- **обнаружение сервисов**: все сервисы регистрируются в Eureka
- **внутреннее взаимодействие**: через Feign-клиенты:
    - event-service → user-service (получение пользователей), request-service (запросы), stats-server (статистика)
    - comment-service → event-service (проверка событий), user-service (пользователи)
    - request-service → event-service (проверка событий), user-service (пользователи)
    - все сервисы → stats-server (для хитов просмотров)
- **база данных**: каждый сервис имеет свою бд
- **конфигурация**: через Config Server, каждый сервис импортирует конфиг с `spring.config.import=configserver:`

## Внутренний API для взаимодействия сервисов

внутренний апи реализован через Feign-интерфейсы в модуле **core/interaction-api/src/main/java/ru/practicum/feign/**

1. **EventClient** (event-service):
    - `GET /events/feign/{eventId}`: получить событие по ID
    - `GET /events/feign/{userId}/{eventId}`: получить событие для пользователя

2. **RequestClient** (request-service):
    - `GET /requests/feign/confirmed?eventIds={list}`: получить подтвержденные запросы для списка событий

3. **UserClient** (user-service):
    - `POST /admin/users`: создать пользователя
    - `DELETE /admin/users/{userId}`: удалить пользователя
    - `GET /admin/users?ids={list}&from={}&size={}`: получить список пользователей
    - `GET /admin/users/{userId}`: получить пользователя по ID

4. **StatsClient** (stats-server):
    - `POST /hit`: добавить хит (просмотр)
    - `GET /stats?start={}&end={}&uris={list}&unique={bool}`: получить статистику

## Внешний API

внешний апи доступен через шлюз gateway-server на порту 8080

апи разделено на:
- **админ API**: `/admin/**` (категории, компиляции, события, пользователи, комментарии)
- **приватный API**: `/users/{userId}/**` (события, запросы, комментарии)
- **публичный API**: `/events/**`, `/compilations/**`, `/categories/**`, `/comments/**`
